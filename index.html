01 — Executive Summary

Motivation was never the problem.
Understanding behavior was.

Habit apps have been around for a decade and most still don't do much more than count. Habitly reasons about your behavior using your own data instead of handing you generic advice.

The Problem

$4.2B habit app market dominated by glorified checklists. High D7 churn (avg 73%) because apps track without understanding. Users get streaks, not insights.

The Vision

An AI-native behavioral system that reads your patterns, coaches your decisions, and evolves with your data. Intelligence embedded at every user action, not bolted on as a feature.

North Star Metric

D30 habit completion rate. DAU and session counts don't tell you if someone actually built a habit. Completion rate does, and that's the whole point of a habit app.

Business Impact

AI-driven personalization increases D30 retention by 2.4× vs static apps. Shareable streak cards create viral acquisition loops with zero paid spend.

02 — Problem Discovery

Habit apps have a
thinking problem.

After analyzing the top 12 habit apps, interviewing 30+ users, and reviewing behavioral science literature, the pattern was clear.

01

Track without reasoning

Apps record completions but never ask why. Skip data exists but is never analyzed. Users get graphs of failure, not guidance to fix it.

High Impact
02

Generic reminders

Notifications fire at a time the user set on day 1. No awareness of current streak status, completion patterns, or which habits are actually at risk today.

High Impact
03

Identity-free habits

Apps track actions, not identity. James Clear's core insight, that habits are identity votes, is universally ignored. Users do habits instead of becoming them.

Medium Impact
04

Cold start failure

First-time users face a blank dashboard. No guidance, no habit suggestions, no behavioral scaffolding. Day 1 is the highest churn moment and apps ignore it.

High Impact
05

Data silos

Mood, habit completion, skip reasons, and energy levels are all tracked separately. No product connects them to surface behavioral correlations.

Medium Impact
06

No narrative layer

Raw data without story is noise. Users see completion percentages but never get a synthesized narrative: "Here's what your week actually looked like behaviorally."

High Impact
Root Cause: Most habit apps get built by engineers optimizing for what's easy to ship, not by anyone thinking hard about behavioral loops. The feature set shows it.

07 — AI Strategy

AI embedded in behavior,
not bolted on.

Model Selection: Why Groq

Speed

Sub-500ms inference vs 2-3s on OpenAI. Habit apps are used in 30-second micro-sessions. Latency kills the experience.

Cost

Free tier sufficient for MVP validation. Groq's pricing allows 10× more AI calls at the same cost as OpenAI.

Architecture

Single Vercel Edge Function proxy. Swapping Groq for OpenAI or Anthropic only touches one file. Provider-agnostic by design.

Model

llama-3.3-70b-versatile: 70B parameters, strong instruction following, excellent at behavioral science framing.

AI Decision Pipeline

User Data Snapshot
Habits, streaks, moods, skips, completionTimes
Context Injection
Structured prompt with behavioral data
Edge Proxy
CORS + API key protection + format conversion
Groq Inference
llama-3.3-70b, <500ms
User Response
Personalized, data-grounded, actionable

AI Guardrails

Prompts constrained to user's own data, no external advice injection
Daily briefing cached client-side to prevent hallucination drift across the same day
JSON schema enforcement for structured AI outputs (Smart Insights, Habit Generator)
Fallback UI if AI call fails, app remains fully functional without AI
No PII sent to model, only behavioral patterns and anonymized stats
All data stays in localStorage, so there's zero server-side data storage

Prompt Engineering

System prompt pattern
You are an elite habit coach.
User data (live):
- Streak: {streak}d
- 7-day rate: {rate}%
- Weakest day: {weakDay}
- Last mood: {mood}/5
- Identity: "{identity}"

Be direct. Reference their
actual numbers. Max 2 sentences.
No generic advice.

The proxy converts Anthropic-format payloads from the frontend to Groq's OpenAI-compatible API. Swapping providers = changing 2 lines in one file.
01

localStorage over database

Zero latency reads. No auth surface. No server cost. Validates core behavioral loop before adding infrastructure complexity. Right tradeoff at MVP stage.

02

Edge Function over backend API

Runs at Vercel's edge globally. Protects the API key without a server. Handles CORS. Zero cold start. Deploys with the frontend.

03

Single shared ai.ts utility

All 9 AI features call one function: askAI(prompt, system). No duplicated fetch logic, and swapping providers means changing one function instead of nine call sites.

13 — Lessons Learned

What building this
actually taught me.

01

AI placement is a product decision, not a feature decision

The hardest question wasn't "should we add AI?" It was "which user action creates the most value when AI intervenes here?" Skip reasons create more coaching leverage than habit creation because friction is higher signal than intent.

02

Architecture decisions are PM decisions

The single-file proxy pattern wasn't really an engineering choice, it was a hedge against vendor risk. Staying provider-agnostic means I'm not stuck with whatever pricing or rate limits one model provider decides to set.

03

localStorage-first was right for now

Every backend I didn't build saved weeks of auth, database, and API design time. That time went into the AI features that actually differentiate the product. Complexity should earn its place.

04

Data you collect but never use is a liability

completionTimes was in the type system from day one, even though nothing used it yet. I knew Best Time Detector would need it eventually, and it turned out to be worth the extra field.

05

Prompt quality is product quality

The difference between a mediocre AI response and a great one had nothing to do with the model. It came down to the prompt. "Be direct. Reference their actual numbers. Max 2 sentences. No generic advice." That constraint is a product decision.

06

Ship to learn, not to show

The LinkedIn post wasn't the goal. The deployed product is a learning instrument. Every day it's live is a day I learn what behavioral loops actually work vs which ones I thought would work.
